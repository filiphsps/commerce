Surface 10 · /cart

Cart

The cart is redrawn as a clean two-column page — line items on the left, a sticky order summary on the right. Each line's variants stop being a bespoke name·value pill and instead reuse the SAME option primitives as the product card and buy box (.opt-swatch for color, .opt-chip for size / weight) as read-only indicators — one swatch system everywhere. The whole line is tenant-customizable through new --cart-line-* tokens on ShopThemeTokens, so the admin theme editor renders the knobs for free.

1 Variant display — before / after

Today cart-line.tsx renders each selected option as a one-off text pill ({name}·{value} in a rounded-md border bg-surface-2 chip) — a styling that exists nowhere else, so a "Sand" swatch on the product card becomes flat text in the cart. The fix reuses the shared primitives read-only: color as an active .opt-swatch + label, size / weight as .opt-chip. Same components the card and buy box use, marked aria-disabled so they read as indicators, not controls.

Both are driven entirely by --cart-line-* tokens added to ShopThemeTokens alongside the 90-odd product-card knobs — so the admin theme editor renders sliders / swatches / toggles for them automatically, no new editor code.

5 Notes

⊕
One swatch systemColor swatches and size / weight chips on the cart line are the exact same .opt-swatch / .opt-chip primitives the product card and buy box render. One option system across card, buy box, and cart — change the swatch design once, it updates everywhere.
i
Tokens → admin for freeThe cart-line look is new --cart-line-* tokens on ShopThemeTokens, declared the same way as the product-card knobs. The admin theme editor introspects the token schema, so it renders the image-size slider, density toggle, divider-color picker, show-vendor / show-sku switches, and variant-style selector with no editor changes.
✓
Read-only semanticsReusing the option primitives in the cart does not make them interactive: each carries aria-disabled="true", drops aria-pressed, and has no click handler. They are visual indicators of what was bought, not pickers — the swatch / chip look is shared, the behavior is not.
