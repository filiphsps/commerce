Storefront Frontend Overhaul

Visual reference for the storefront fix pass. These are static, token-driven mockups — they re-skin from tokens.css exactly the way a tenant theme re-skins the real storefront, and they compose one shared component layer (ui.css) so the spec itself proves the DRY direction. Iterate here before any app code changes.

◆ Design direction

Hybrid (locked): keep the platform default tokens, accent palette, and Public Sans. Reimagine layout only where it is broken or weak. No tenant theme is invalidated — every fix is expressed as a token or a layout change that any theme inherits.

⊕ DRY consolidation

⊕
One option system everywhere.opt-swatch / .opt-chip / .opt-more render variants on the product card, the buy box, AND cart lines. Weight / color / size all flow through the same primitives.
⊕
One filter, three pagesThe faceted filter component serves /products, collection pages, and search. Same facets, same chips, same mobile drawer.
⊕
One railThe carousel and the grid are one component in two modes — gutter padding fixes the clipped edge shadow for both.
⊕
Tokens, not hard-codesCart-line look is new --cart-line-* tokens added to ShopThemeTokens, so the admin theme editor renders the knobs for free. The dark-bg fix is a token flip (.on-dark), not per-component overrides.
