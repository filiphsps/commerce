Surface 04 · /collections/[handle]

Collection page

The route renders a bare <Heading title> with no treatment for the collection's cover image, so a collection with an image gets no hero and one without looks weak and empty. Two header treatments fix both ends — a scrimmed hero band when there's an image, a confident typographic header when there isn't. The product grid then reuses the shared .pc-grid auto-fill so cards fill every row evenly from phone to ultrawide.

1 Header — two cases

Today the header is a centered <Heading> regardless of whether the collection has a cover image. With no image it reads thin; with an image the image is simply dropped. Below: the no-image header fixed as a typographic block (left/right), then the full hero treatment for the with-image case.

2 Grid fill

"Cards look weird at mobile and some resolutions" is the grid not filling evenly. The shared .pc-grid uses repeat(auto-fill, minmax(var(--product-card-min-width), 1fr)) — cards clamp to a sensible minimum, then the 1fr share distributes the remainder so a row is always full and no card stretches to an absurd width or leaves a dead gap. One toolbar (count · sort · filter trigger) sits above it.
