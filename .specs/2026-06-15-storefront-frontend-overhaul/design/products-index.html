Surface 06 · /products + shared filter

All products & filters

The /products route exists but today exposes only a vendor select and a sort select — you cannot actually narrow the catalog. This adds a full faceted filter (price, availability, vendor, product type, and product options as color swatches + size chips) and, because it is the DRY anchor, the same component is reused by collection pages and search. One filter, three pages.

1 Before / after

Before: a thin bar with a sort select and a vendor select — the only two ways to slice the catalog. After: a real sidebar of facets (the locked set) beside a results grid, with active-filter chips and a mobile drawer. The facets are derived from Shopify's product filters + variant options, so they reflect what is actually in the catalog.

2 The page

The shared filter lives in .shop-aside > .filters as a stack of <details class="facet"> groups (open by default, no JS needed). Results sit beside it: a .toolbar with the count + sort + a mobile .filter-trigger, an .active-filters row of removable .filter-chips, and the one true .pc-grid of .pc cards. Every selection is mirrored to the URL.

3 Mobile filter drawer

On phones the sidebar collapses (.shop-aside is display:none under 900px) and the .filter-trigger in the toolbar opens the SAME facets as a bottom sheet — pinned to the bottom of the viewport, scrollable, with a sticky footer that applies the current result count and a clear link. No new facet markup: the drawer renders the identical .facet groups.

4 Why this is the DRY anchor

⊕
One filter, three pagesThis exact component — .filters sidebar, .active-filters chips, and the bottom-sheet drawer — serves /products, every collection page, and search. Same facets, same chips, same mobile behavior; only the result set and the available facet values differ.
i
URL-param drivenEvery selection is reflected in the query string: ?vendor=&type=&color=&size=&price=&availability=&sort=. State lives in the URL, so filtered views are shareable, back/forward works, and the product-card vendor link (/products?vendor=acme-studio) deep-links straight into a pre-selected facet.
✓
Facets come from the dataAvailable facets and counts are derived from Shopify product filters and variant options — not a hand-maintained list. Color renders as .opt-swatch, size as .opt-chip, and an option value with no in-stock match shows as .unavailable rather than disappearing.
