Surface 03 · product-card

Product card

There is exactly one card and one option system. The same .pc shell and the same .opt-swatch / .opt-chip primitives render in the grid, the buy box, and the cart line. This page documents the card's anatomy and states, the locked vendor-link target, and the dark-background selected-state fix — one token flip that corrects every component at once.

1 Anatomy & states

Every card is a .pc with .pc-media (badges + float-pill CTA), then .pc-body holding the .pc-vendor eyebrow, .pc-title, .pc-price, and an optional .pc-options row. The states below are the same component under different data — never different components.

2 Vendor / brand link

The .pc-vendor eyebrow is a real link to the brand. Its target resolves at render time so a shop with curated brand collections gets the rich landing page, while everyone else still gets a working, filtered listing.

The card never renders a dead link.
⊕
Same resolver everywhereThe buy box vendor link and any "Shop by brand" list resolve the target through the same helper — one resolution order, not three copies.

3 Selected color on dark backgrounds

On a dark surface — the homepage banner, a dark-themed section, or a dark product-card area — the default selected-swatch ring (--product-card-swatch-ring-color: #14110b) and the active chip (dark ink on white) vanish into the background. The fix is NOT a per-component override: adding .on-dark flips the ink/border/ring tokens so every component below reads correctly at once.

CauseThe selected ring color is a hard-coded dark token (--product-card-swatch-ring-color: #14110b) and the active chip is dark ink on white. Both disappear against a dark surface — invisible or near-invisible selection.
✓
FixComponents read tokens, never hard-coded ink. .on-dark flips the ring to #ffffff and the active chip to a white background with ink text. One token flip on the dark wrapper — no per-component hacks, no extra props.
⊕
One flip fixes allBecause cards, the buy box, and cart lines all read the same swatch/chip tokens, the single .on-dark context the homepage banner already uses fixes selection contrast across every surface at once.
