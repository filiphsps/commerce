Surface 05 · /collections (new route)

Collections index

Today only /collections/[handle] exists; the bare /collections route 404s. This surface introduces it: a browsable, token-driven listing of every collection in the shop, built from the same .container + grid conventions as the rest of the spec.

1 The page

A hero header (eyebrow + title + short description) over a responsive grid of collection cards. The first two run as larger editorial "featured" tiles (a two-track span); the rest fill an auto-fill minmax(280px, 1fr) grid. Each card carries a product-count badge. Colors come straight from the locked accent palette via tokens.css — no hard-codes.

2 Anatomy of a collection card

The card is intentionally lean — a media area, a title, and a count. It mirrors the product card's shape (same radius, border, shadow, and hover lift tokens) so the two read as one family across the storefront.
