Surface 01 · /products/[handle]

Product page

The gallery is sound in structure (sticky thumbnails + cross-fading primary image) but the fullscreen lightbox leaks: a Tailwind flex utility on the native <dialog> overrides the UA dialog:not([open]){display:none} rule, so it renders permanently — floating over and under siblings.

1 The lightbox bug

Root cause in product-gallery-lightbox.tsx: the dialog className starts with fixed inset-0 m-0 flex …. flex wins over the user-agent dialog:not([open]){display:none} by specificity, so the dialog is always laid out. It is not in the top layer (because showModal() only ran on open), so it stacks against page z-index — hence "over some elements and under others".

CauseUtility flex overrides dialog:not([open]){display:none}. The dialog is rendered unconditionally and never enters the top layer until opened.
✓
FixKeep the dialog display-none when closed and only flex when open: gate the flex with open:flex [&:not([open])]:hidden (or render-on-open). The native top layer then handles stacking — no z-index war, no bleed.
