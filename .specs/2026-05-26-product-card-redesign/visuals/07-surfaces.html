Bug Root cause Fix
Search renders 0 cards despite "3 PRODUCTS" Hypothesis: firstAvailableVariant returns null OR a server-component error swallowed by Suspense. Needs root-causing. Add smoke test (cards.length === resultsCount) + investigate render path in spec phase 1.
"+0" overflow chip rendering Chip text is computed separately from data-overflow attr; chip element renders regardless. :global([data-overflow="0"]) CSS rule never fires. Conditional render in JSX: don't render chip when overflow is 0. Drop the CSS hide rule entirely.
Yellow/cream banding inside image tile object-fit: contain + product images with mismatched aspect → wrapper bg shows through. Default to object-fit: cover. Tenants opt-in to contain via --product-card-image-fit token.
Carousel overflows viewport, no scroll affordance No carousel primitive — just a flex/grid container with overflow. New RecommendationsRail: scroll-snap, edge-fade, arrows on desktop hover.
Size pills with text strikethrough Sizes rendered on base card as pills; unavailable shown as text-strikethrough. Drop sizes from base card. Sizes only in picker. Unavailable = pill with CSS strike line (no text-decoration).
Out-of-stock cards drop to 50% opacity Chassis hard-codes opacity-50. Token --product-card-oos-opacity (default 0.7).
