Why pseudo-strike

text-decoration: line-through draws at the font baseline and looks bad with tabular-nums and proportional currency symbols. A pseudo-element strike sits exactly through the optical center of the numerals and respects line-height.

Accessibility — chassis-layer rules

  • Hit targets: all interactive elements ≥ 36×36px (clears WCAG 2.5.5 AAA). Visual swatch is 16px; the rest is padding inside a 36px button.
  • Focus visible: 2px outline at 2px offset, color --accent. Same treatment on every interactive primitive.
  • ARIA labels: every icon-only control has aria-label. Swatches use the color name; CTA uses "Quick add" or "Add to bag · $X" with the resolved price.
  • Picker focus management: trap focus inside; ESC and backdrop click close; restore focus to the triggering CTA on close.
  • State announcements: live region announces variant swap ("Color changed to Brick") and cart additions ("Added to bag · Bone size M"). Today's swapAnnouncement pattern in cart-line.tsx is the model.
  • Contrast: default tokens clear WCAG AA on the default tile bg. Tenant overrides need their own audit — spec includes a token-validation helper that warns in dev when a token override drops below AA.
  • Reduced motion: drops transforms; opacity-only fades remain.
  • Keyboard: tab to swatches, Enter/Space to select; tab to CTA, Enter/Space to open picker; arrow keys inside picker to traverse chips (managed by existing ProductOptions primitives).
