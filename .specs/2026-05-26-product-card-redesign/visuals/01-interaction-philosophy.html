Which philosophy do you want the card to follow?

A

Mini-PDP, but execute it properly

Size pills + color swatches + Add-to-bag stay in the card. We fix every defect (no +0 leak, no strikethrough text, consistent option treatment, fixed heights, proper image fit).

B

Browse tile, options live on the PDP

Card = image, vendor, title, price, color hint, hover "Quick add". Click goes to PDP for the buy. Same shape works everywhere.

C

Hybrid: browse default, mini-PDP on long-press / hover-expand

Default is the browse tile. A hover-overlay or long-press reveals size pills + add-to-cart without changing the card's footprint in the grid. Best-of-both, more complexity.

Note: real bugs I confirmed in DOM (search renders 0 cards, +0 chip text wrong, carousel overflows, image banding) are getting fixed regardless of which path. The choice is about where variant selection lives.
