18 · Accessibility · chassis-layer rules
  • Hit targets: ≥ 36×36 px (clears WCAG 2.5.5 AAA). Visual swatch 16px; the rest is hit padding.
  • Focus visible: 2px outline at 2px offset, color --accent. focus:outline-none + focus-visible:outline-… on every interactive primitive.
  • focus-within on card chassis: shadow lifts when any descendant is keyboard-focused — keyboard users locate the active card across a grid.
  • Selection: user-select: none on interactive primitives, user-select: text on title/price/vendor blocks.
  • Touch: -webkit-tap-highlight-color: transparent + touch-action: manipulation (kills 300ms tap delay and zoom).
  • ARIA: aria-label on every icon-only button. aria-pressed on toggles. aria-expanded + aria-controls on CTA when picker open. aria-haspopup="dialog" when routing to sheet.
  • Title link as canonical focus: image-overlay PDP link is aria-hidden="true" tabindex="-1" — one focus target per destination.
  • Live regions: aria-live="polite" for variant swap and cart-add announcements.
  • Picker: Sheet uses Radix Dialog wrapper (focus trap, scroll lock, ESC/backdrop close, focus restoration). Float uses Radix Popover wrapper (focus trap, ESC/outside-click). Inline is in-page expansion with local focus management.
  • Reduced motion: all transitions wrapped in motion-safe: Tailwind variant.
  • Forced colors: borders preserved via forced-colors: variants; focus rings remain visible.
