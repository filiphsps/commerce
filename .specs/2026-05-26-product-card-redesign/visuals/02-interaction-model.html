Decisions captured in the demo

Base card
  • One CTA entry: the "+" affordance over the image. No "Add to bag" full-width button on the card itself.
  • Color swatches are a display hint; click swaps image + pre-selects color in ProductOptions.Root.
  • No size pills on the base card.
  • Single buyable variant: green dot on "+" + label change to "Add to bag" — "+" adds directly.
  • Card body / title both link to PDP.
Picker (float OR sheet, same contents)
  • Shows only options that need picking. On vertical-float, color is already chosen → picker shows Size only. On sheet, full Size + Color shown (no swatch row at that scale).
  • Add-to-bag CTA inlines the resolved variant price.
  • Unavailable values are chips with a strike line drawn as a CSS rule — never with text strikethrough.
  • Float picker is anchored bottom-of-image; closes via "+" (now an "×") or click outside.
  • Sheet picker is centered dialog on desktop, bottom sheet on mobile (drag-handle visible).
Surface routing (locked from the "Recommended" answer)
  • Vertical card on a grid → float picker
  • Horizontal card (search row, cart drawer) → sheet picker
  • Viewport < 768px → sheet picker, regardless of card layout
  • Cart-drawer card: no "+" at all (read-only)
  • Tenant token --product-card-quick-add-presentation can pin to float or sheet
