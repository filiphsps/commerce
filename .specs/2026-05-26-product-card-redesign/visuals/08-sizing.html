Bounded card width — v2 (frontend-design pass)

Goal: card always feels like a product thumbnail at every viewport. Density beats size at scale.

Reframing the problem

What changed in my thinking
  • Previous fix capped card at 320px and added a horizontal switch above 360px. That's still too loose — the card can hit 320px in sparse grids and the image becomes huge.
  • Reframed: A card is a thumbnail, not a hero. At wider viewports the user wants more products visible, not bigger products visible. Density beats size.
  • New target: minmax(200px, 260px) — a tight range. Image stays a sensible size; type stays readable; brand identity is consistent at every viewport.
  • Page container caps the grid. Site already has --page-width (e.g. 1280px); grid lives inside it. Cards never get absurd because the container doesn't.
  • Sparse data = empty space, not stretched cards. 3 items on a wide viewport sit left-aligned with empty space on the right (or centered, tenant choice). Three balloon-cards is broken; three intentional thumbnails is design.
  • Dropped the horizontal switch. With 260px max it never fires. Code path no one wants.
