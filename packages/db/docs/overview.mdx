The typed data-access seam for the Nordcom Commerce data store. Every app reads and
writes shops, users, sessions, identities, reviews, and feature flags through the
service instances exported here; the services translate those calls onto the Convex
deployment (`packages/convex`). No other workspace talks to the Convex `db/*`
functions directly.

> **Server-only.** The package is marked `'server-only'`. The Convex clients are
> constructed lazily, but calling any service without `CONVEX_URL` (and, for the
> identity-less paths, `CONVEX_SERVER_SECRET`) in the environment will throw.

For the canonical reference, see the [README on GitHub](https://github.com/filiphsps/commerce/blob/master/packages/db/README.md).
