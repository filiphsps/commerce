React integration layer for the cart system ÔÇö a context provider that manages the
optimistic mutation queue, slice contexts for granular re-render control, and hooks
that expose the cart surface to client components.

## Architecture

`@nordcom/cart-react` owns the client-side state model:

- **`CartProvider`** ÔÇö the root provider. Accepts a `KernelSnapshot` (initial cart
  state from the server) and a `dispatch` function (a Next.js server action or any
  `(mutation) => Promise<CartActionResult>` callable). Holds the optimistic queue
  in a `useReducer` and fans state out across all slice contexts.
- **Slice contexts** ÔÇö `CartCountContext`, `CartLinesContext`, `CartCostContext`,
  `CartMetaContext`, `CartStatusContext`, `CartPendingContext`. Each context carries
  exactly one slice, so components that only care about the count do not re-render
  on cost changes.
- **Hooks** ÔÇö `useCartCount`, `useCartLines`, `useCartCost`, `useCartMeta`,
  `useCartStatus`, `useCartPending`, `useCartActions`, `useCartDispatch`. Thin
  wrappers around `useContext` with narrow return types matching their slice.
- **`CartPredictor` / `LinePredictor`** ÔÇö optional callback pair that the host
  supplies to project optimistic cost and line state before the server round-trip
  returns. Keeps totals non-stale during rapid mutations.
- **`ClientAuthBridge`** ÔÇö client-side counterpart to `AuthBridge`; resolves buyer
  identity to merge on `update-buyer-identity` mutations dispatched from the client.

## Optimistic queue

Every call to `dispatch(mutation)` follows this path:

```
dispatch(mutation)
  Ôćĺ idempotency key generated (crypto.randomUUID)
  Ôćĺ optimistic prediction applied via CartPredictor
  Ôćĺ PendingMutation appended to queue Ôćĺ UI re-renders immediately
  Ôćĺ server action called in background (useTransition)
  Ôćĺ confirmed result merges into projection Ôćĺ pending entry removed
```

The `stale` flag on `CartCostValue` is `true` while at least one predicted mutation
has not been confirmed, letting UI indicate that totals are estimates.

## Usage

```tsx
import { CartProvider } from '@nordcom/cart-react';

export default function Layout({ snapshot, dispatch, children }) {
    return (
        <CartProvider snapshot={snapshot} dispatch={dispatch}>
            {children}
        </CartProvider>
    );
}
```

```tsx
import { useCartCount, useCartActions } from '@nordcom/cart-react';

function AddToCartButton({ variantId }) {
    const count = useCartCount();
    const { addLines } = useCartActions();

    return <button onClick={() => addLines([{ merchandiseId: variantId, quantity: 1 }])}>{count}</button>;
}
```
