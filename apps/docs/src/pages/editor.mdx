# Building the admin editor

The Nordcom Commerce admin's content-editing surface — every "edit page",
"new page", list, versions view — is generated from a single source of
truth per collection: an **editor manifest** that lives alongside the
Payload `CollectionConfig` for that collection.

A manifest is ~30 lines of TypeScript. A per-route `page.tsx` is ~25 lines
of React. Everything between — form chrome, draft/publish toolbar,
autosave debounce, locale switcher, live-preview iframe, server actions,
revalidation paths — comes from the shared `@nordcom/commerce-cms/editor`
primitives.

## The mental model

```
Payload CollectionConfig       Single source of truth for fields, drafts,
                               versions, localization, hooks.

CollectionEditorManifest       Adds route metadata, access predicates,
                               list columns, live-preview URL, and
                               revalidation paths on top of the collection.

EditorRuntime                  Per-app dependency bundle: auth context,
                               form-state builder, shell components.

<EditorEditPage>               Server components that consume manifest +
<EditorListPage>               runtime + route params and render the right
<EditorNewPage>                Payload form / table / version list inside
<EditorVersionsPage>           the admin's chrome.

pnpm cms:gen                   Codegen step that emits 'use server' action
                               wrappers per manifest. CI checks they're in
                               sync via pnpm cms:gen:check.
```

## Where the code lives

| Concept | Path |
|---|---|
| Editor primitives | `packages/cms/src/editor/` |
| Manifests | `packages/cms/src/editor/manifests/<slug>.ts` |
| Admin runtime | `apps/admin/src/lib/editor-runtime.ts` |
| Generated actions | `apps/admin/src/lib/cms-actions/_generated/<slug>.ts` |
| Per-route page.tsx | `apps/admin/src/app/(app)/(dashboard)/[domain]/<area>/<slug>/page.tsx` |
