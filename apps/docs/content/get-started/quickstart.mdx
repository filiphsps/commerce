Nordcom Commerce is a multi-tenant headless e-commerce platform. A single Next.js
deployment serves an arbitrary number of shops resolved by hostname.

## Prerequisites

- Node.js (version pinned in `.nvmrc`)
- pnpm
- A running MongoDB instance

<Callout type="tip">
    Run `pnpm dev:storefront` if you only need the storefront — it boots a single app
    and skips the admin and landing builds. Use `pnpm dev` to start all three in parallel.
</Callout>

## Install

```bash
pnpm install
cp .env.example .env       # set MONGODB_URI, AUTH_SECRET, SERVICE_DOMAIN at minimum
pnpm build:packages        # apps import from each package's dist/, not source
pnpm dev                   # all apps in parallel
```

| App | URL |
| --- | --- |
| Storefront | https://storefront.localhost |
| Admin | https://admin.localhost |
| Landing | https://landing.localhost |
| Docs | https://docs.localhost |

<Callout type="note">
    `pnpm dev` boots a persistent local MongoDB under `.mongo-dev/` via
    `@nordcom/commerce-test-mongo`. Set `MONGODB_URI` in your shell to use an
    external instance instead. Run `pnpm dev:reset` to tear down the local instance.
</Callout>

## Seed a tenant

The storefront resolves every request by hostname. Without a matching `Shop` record in
MongoDB, every request redirects to `/status/unknown-shop/`. In development, unknown
hosts fall back to `nordcom-demo-shop.com` automatically so you can boot without seeding.

To add a real tenant, insert a row into the `shops` collection:

```ts
await db.collection('shops').insertOne({
    domain: 'myshop.com',
    name: 'My Shop',
    commerceProvider: { type: 'shopify', domain: 'myshop.myshopify.com', token: '...' },
    contentProvider: { type: 'shopify' },
    defaultLocale: 'en-US',
});
```

No redeploy required. Middleware picks up the new row on the next request.

## Quality gates

Before pushing, run:

```bash
pnpm lint        # Biome lint
pnpm typecheck   # TypeScript across all packages and apps
pnpm test        # Vitest unit tests (requires MONGODB_URI)
```

CI runs the same gates on every PR.
