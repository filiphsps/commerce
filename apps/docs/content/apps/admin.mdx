---

`apps/admin` is the back-office app. It embeds `@nordcom/commerce-cms` (Payload 3.x)
and provides the CMS admin UI, tenant management dashboard, media library, and the
authentication server that the storefront shares via NextAuth.

## Entry point

`src/proxy.ts` enforces authentication via NextAuth and handles one back-compat redirect:
legacy `/cms/*` URLs (from the old Payload-mounted admin shell) are rewritten to the
dashboard root so existing bookmarks continue to work.

## Application structure

| Route group | Purpose |
| --- | --- |
| `(app)/` | Authenticated dashboard — shops, tenants, users, media |
| `(payload)/` | Payload CMS admin UI, mounted at the app root |
| `api/` | Auth endpoints (NextAuth) and any JSON API routes |

<Callout type="note">
    CMS routes for a specific shop live at `/[domain]/content/…` inside `(app)/`.
    Tenant-level routes (shops, users, media) live at the path root. Both are accessible
    once authenticated.
</Callout>

## CMS integration

The admin app embeds the CMS package via `src/payload.config.ts`. All content collections
— pages, articles, navigation, product overlays — are defined there. After any change to
a collection field, regenerate the Payload types:

```bash
pnpm --filter @nordcom/commerce-cms generate:types
```

The storefront consumes Payload content through the Local API (`@nordcom/commerce-cms`
is `server-only`). The admin app is the only runtime that can write to Payload collections
directly.

## Authentication

NextAuth is configured in `src/utils/auth.config.ts`. It supports:
- **GitHub OAuth** — for staff/developer login.
- **Credentials** — email/password for shop owners.

The storefront reads auth state from the same MongoDB collection via `@nordcom/commerce-db`
(`UserService`). Both apps must share the same `NEXTAUTH_SECRET` and `NEXTAUTH_COOKIE_NAME`.

## Key environment variables

| Variable | Purpose |
| --- | --- |
| `MONGODB_URI` | MongoDB connection string |
| `PAYLOAD_SECRET` | Payload CMS signing secret |
| `NEXTAUTH_SECRET` | Session token signing secret — must match storefront |
| `SERVER_URL` | Public URL of this admin app (for OAuth callbacks, S3 media URLs) |
| `GITHUB_ID` / `GITHUB_TOKEN` | GitHub OAuth credentials |
| `S3_ENDPOINT` / `S3_BUCKET` / `S3_ACCESS_KEY_ID` / `S3_SECRET_ACCESS_KEY` | S3 media storage |
